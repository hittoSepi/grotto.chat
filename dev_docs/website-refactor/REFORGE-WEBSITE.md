GROTTO asset usage plan
1. Brand hierarchy

Tämä on tärkein sääntö:
Primary brand

Logo + wordmark

    käytä navigaatiossa

    hero-otsikon yhteydessä

    favicon/app icon -linjassa

    download-sivuilla

    docs-headerissa

Secondary brand

Cave / tunnel visuals

    landing page hero tausta

    section taustat

    installer splash

    security / hosting osiot

Supporting mascot

Grotto daemon

    onboarding

    loading / empty states

    docs-kuvitus

    footer easter egg

    installer / terminal / CLI

    joskus security/hosting callout

Eli lyhyesti:

    Logo myy tuotteen.
    Luola luo maailman.
    Daemon tuo persoonan.

Jos daemon nousee pääosaan väärässä paikassa, koko homma alkaa näyttää peliltä eikä turvalliselta chat-appilta.

3. Landing page placement plan
Header

Käytä:

    pieni wordmark tai icon + wordmark

Älä käytä:

    daemonia

    täyttä luolalogo-lockupia isona

Headerin pitää olla siisti ja nopea lukea.
Hero
Suositus:

    vasemmalla headline, supporting text, CTA:t

    oikealla grotto-black.png tai grotto-transparent.png

    taustalla kevyt cave texture / signal glow

Optional:

    pieni terminal-style status panel:

        secure tunnel ready

        voice caverns online

        admin tui available

Älä tee:

    hero = pelkkä daemon portrait

    hero = liikaa lorea

    hero = kolme eri assettia yhtä aikaa

Heroon yksi päävisual riittää.
Why GROTTO

Käytä:

    ei välttämättä mitään isoa kuvaa

    ehkä hyvin pieni cave/tunnel motif

    kortit hoitavat työn

Käytä:

    luolakuva tai cave background panel

    copy: self-hosted servers, admin TUI, optional public directory

Miksi:

Clients / Platforms

Tässä käyttäisin mieluummin:

Tähän voi laittaa pienen daemon calloutin jos haluat.

    liian painavaa kuvaa

5. Käytännön asset rules
Rule 1

Yhdellä näkymällä vain yksi “tähtiassetti”.

Esim:

    Hero: logo-lockup

    Security: daemon portrait

    Hosting: cave art

Ei kaikkia samaan ruutuun.
Rule 2

Daemon näkyy korkeintaan 1-2 kertaa per sivu.

Muuten siitä tulee maskottikampanja eikä tuotebrändi.
Rule 3

Logo aina ensin.
Jos pitää valita logo tai daemon, valitse logo.
Rule 4

Cave art = mood
Daemon portrait = personality
Logo = identity

Pidä nämä roolit erillään.

7. Mitä pyytäisin Codexia tekemään assettien osalta

Voit lisätä promptiin tämän lisäohjeen:

Asset usage guidance:
- Use the main GROTTO logo lockup as the hero visual.
- Use the daemon portrait only as a secondary/supporting visual, ideally in the Security section or a small supporting callout.
- Use the cave illustration in the Hosting / “Run your own cave” section or as a dark atmospheric supporting panel.
- Keep the daemon limited to 1–2 appearances on the page max.
- Do not use the daemon as the main logo or primary hero identity.
- Footer may include a subtle “Approved by one domestic grotto daemon.” easter egg if it fits the tone.
